# 第六章：A2A协议详解

## 6.1 A2A 协议概述

A2A (Agent to Agent) 是一种用于 Agent 之间通信的协议，支持多 Agent 协作和分布式任务处理。

```
┌──────────────────┐                         ┌──────────────────┐
│    A2A Client    │◄──────────────────────►│   A2A Server     │
│                  │                         │                  │
│  - AgentExecutor │                         │  - SkillHandler │
│  - TaskClient    │                         │  - TaskServer   │
└────────┬─────────┘                         └────────┬─────────┘
         │                                           │
         │  HTTP/JSON-RPC                           │
         └───────────────────────────────────────────┘
                   │
                   ▼
         ┌─────────────────────┐
         │    A2A Protocol     │
         │  - Task Protocol   │
         │  - Skill Discovery │
         │  - State Transfer  │
         └─────────────────────┘
```

## 6.2 A2A Manus Agent 实现

**文件位置**：`protocol/a2a/app/agent.py`

```python
from app.agent.manus import Manus

class A2AManus(Manus):
    """支持 A2A 协议的 Manus Agent"""

    async def invoke(self, query, sessionId) -> str:
        """处理 A2A 调用请求"""
        config = {"configurable": {"thread_id": sessionId}}
        response = await self.run(query)
        return self.get_agent_response(config, response)

    async def stream(self, query: str) -> AsyncIterable[Dict[str, Any]]:
        """流式响应（A2A 支持但 Manus 暂不支持）"""
        raise NotImplementedError("Streaming is not supported by Manus yet.")

    def get_agent_response(self, config, agent_response):
        """转换 Agent 响应为 A2A 格式"""
        return {
            "is_task_complete": True,
            "require_user_input": False,
            "content": agent_response,
        }

    SUPPORTED_CONTENT_TYPES: ClassVar[List[str]] = ["text", "text/plain"]
```

## 6.3 A2A Server 实现

**文件位置**：`protocol/a2a/app/main.py`

```python
from a2a.server.starlette import A2AStarletteApplication
from a2a.types import (
    Agent card,
    TaskNotFoundError,
    TextPart,
    TaskState,
)
from .agent import A2AManus

# 定义 Agent 技能
skills = [
    Skill(
        id="general_task",
        name="General Task",
        description="Handle general tasks using Manus Agent",
        tags=["manus", "general", "assistant"],
        input_modes=["text", "text/plain"],
        output_modes=["text", "text/plain"],
    ),
]

# 创建 Agent 实例
agent = A2AManus()

# 创建 A2A 应用
app = A2AStarletteApplication(
    agent=agent,
    agent_card=AgentCard(
        name="OpenManus Agent",
        description="A versatile AI agent based on Manus",
        url="http://localhost:5000",
        version="1.0.0",
        skills=skills,
    ),
)

# 添加路由
app.add_route("/agent", agent, restore_api_state=True)

# 运行服务器
if __name__ == "__main__":
    import uvicorn
    uvicorn.run(app, host="0.0.0.0", port=5000)
```

## 6.4 A2A 协议消息格式

### 6.4.1 任务创建请求

```json
{
  "jsonrpc": "2.0",
  "method": "tasks/send",
  "params": {
    "id": "task-123",
    "sessionId": "session-456",
    "message": {
      "role": "user",
      "parts": [
        {
          "kind": "text",
          "text": "帮我写一个 Python 函数"
        }
      ]
    }
  },
  "id": 1
}
```

```json
{
  "jsonrpc": "2.0",
  "result": {
    "id": "task-123",

    "message": {
      "role": "agent",
      "parts": [
        {
          "kind": "text",

```

## 6.5 技能发现机制

```python
@dataclass
class Skill:
    """Agent 技能定义"""
    id: str                           # 技能唯一标识
    name: str                         # 技能名称
    description: str                  # 技能描述
    tags: List[str]                   # 技能标签
    input_modes: List[str]           # 支持的输入模式
    output_modes: List[str]           # 支持的输出模式

    def matches_query(self, query: str) -> bool:
        """检查技能是否匹配查询"""
        query_lower = query.lower()
        return (
            query_lower in self.name.lower()
            or query_lower in self.description.lower()
            or any(query_lower in tag for tag in self.tags)
        )
```

## 6.6 与 MCP 的对比

| 特性 | MCP | A2A |
|------|-----|-----|
| **定位** | AI ↔ 工具/数据 | Agent ↔ Agent |
| **用途** | 扩展 AI 能力 | 多 Agent 协作 |
| **通信** | stdio / SSE | HTTP / WebSocket |
| **发现机制** | 工具列表 | 技能卡片 |
| **状态管理** | 无状态 | 任务状态跟踪 |
| **典型场景** | 文件系统、Git、数据库 | Agent 调度、分布式处理 |

## 6.7 A2A 工作流程

```
┌─────────────┐                    ┌─────────────┐                    ┌─────────────┐
│   Client    │                    │  A2A Server │                    │   Agent     │
└──────┬──────┘                    └──────┬──────┘                    └──────┬──────┘
       │                                  │                                  │
       │  1. 获取 Agent 卡片              │                                  │
       │──────────────────────────────► │                                  │
       │  2. Agent 卡片 (技能列表)        │                                  │
       │◄─────────────────────────────── │                                  │
       │                                  │                                  │
       │  3. 创建任务                     │                                  │
       │──────────────────────────────► │                                  │
       │                                  │  4. 转发到 Agent                 │
       │                                  │ ─────────────────────────────►  │
       │                                  │                                  │
       │                                  │  5. Agent 执行                   │
       │                                  │ ◄─────────────────────────────   │
       │                                  │                                  │
       │  6. 任务状态更新                 │                                  │
       │◄────────────────────────────── │                                  │
       │                                  │                                  │
       │  ... (循环直到完成)              │                                  │
       │                                  │                                  │
       │  7. 获取结果                     │                                  │
       │──────────────────────────────► │                                  │
       │  8. 最终结果                     │                                  │
       │◄─────────────────────────────── │                                  │
```
